The conformance parameters for is used primarily to capture uses case information about the specification. Double click on any of the grids to bring up an edit window for entering larger amounts of text. See a description of the Static and Dynamic Profile Identifiers below the screen image.

This form also displays the ASN1 identifiers associated with the profile. These are automatically derived. The information in the Static Profile is derived from the specification information on the Message Definition tab, while the Dynamic ID is derived from the Message Definition tab and from the Acknowledgement information on the Conformance Profile form. A place holder is also provided for documentation of an HL7 Conformance Registry OID if available.

  • Conformance SIG ID - ConfSig(1)
  • An organization identifier - NATA(1)
  • The HL7 Version - 2.3(5)
  • Profile Type - static-profile(1)
  • Message Type - ORU(23)
  • Event Type - R01(131)
  • Order Control Code - null(0)
  • Structure Type - ORU_R01(53)
  • Specification Version - 1.2(1)
  • Role - Sender(1)
  • Conformance SIG  ID- ConfSig(1)
  • An organization identifier - NATA(1)
  • Profile Type - dynamic-profile(2)
  • Acknowledgement Style -AccNE_AppAL(2)
  • Acknowledgement Mode - defer_mode_ack(1)
