Running the MWB in Debug Mode

 

Occasionally it has been reported that the Messaging Workbench fails to start. The most prevalent cause of this failure to date has been missing or corrupt ODBC files. Running the MWB in debug mode allowed diagnosis of the cause, and a work around has been developed for that particular problem.  If you encounter a problem in starting the MWB, run the application in debug mode and forward the log file created to the Conformance Group representative assisting you with the problem.

To run the MWB in debug mode click on the Windows Start button and select Run...

Use the Browse feature of the Run window (illustrated below) to find the application executable MsgWB.exe and select it.

At the end of the Open field in the window, add /d after the executable path just as shown above. Click the OK button to run the application. Look for the file MWBDebugLog.log in the application directory.
